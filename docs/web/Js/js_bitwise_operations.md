# JavaScript 位运算详解

## 先建立最基础的直觉：计算机里的数字长什么样？

在计算机里，所有数字最终都是以二进制存储的，也就是一串 0 和 1。JavaScript 的位运算操作的是一个整数的 **32 位二进制表示**。

比如十进制的 `13`，它的 32 位二进制是：

```
00000000 00000000 00000000 00001101
```

你可以用 `(13).toString(2)` 在 JS 里验证，得到 `"1101"`，前面的 0 省略了而已。

位运算就是直接操作这 32 个格子里的每一个 0 和 1，不经过任何十进制的转换，所以速度极快，但理解起来需要习惯这种"格子操作"的思维。

## 第一个运算符：`&`（按位与）

`&` 的规则是：**两个对应位都是 1，结果才是 1**，否则是 0。你可以把它理解成"同时满足才算数"。

```js
5 & 3
// 5 的二进制：0101
// 3 的二进制：0011
// 按位对比：  0001  ← 只有最后一位两个都是 1
// 结果是 1
```

`&` 最经典的用途是**提取某一位的值**。比如你想知道一个数字的最低位是 0 还是 1（也就是这个数是奇数还是偶数），你就用 `n & 1`：

```js
7 & 1  // 0111 & 0001 = 0001 = 1，说明最低位是 1，7 是奇数
8 & 1  // 1000 & 0001 = 0000 = 0，说明最低位是 0，8 是偶数
```

这比 `n % 2` 更底层，性能也更好。你在上面 190 题里看到的 `n & 1` 就是这个用途——每次从 `n` 里取出最低位的值。

另一个经典用途是**判断一个数是不是 2 的幂**。2 的幂的二进制表示只有一个 1，比如 `8 = 1000`。而 `8 - 1 = 7 = 0111`，把这两个数按位与，结果一定是 0：

```js
function isPowerOfTwo(n) {
    return n > 0 && (n & (n - 1)) === 0;
}
// isPowerOfTwo(8) → true，因为 1000 & 0111 = 0000
// isPowerOfTwo(6) → false，因为 0110 & 0101 = 0100 ≠ 0
```

## 第二个运算符：`|`（按位或）

`|` 的规则是：**两个对应位只要有一个是 1，结果就是 1**。你可以理解成"有一个满足就算数"。

```js
5 | 3
// 5：0101
// 3：0011
// 结果：0111 = 7
```

`|` 最常见的用途是**给某一位设置为 1**（也叫"置位"）。比如你想把第 2 位（从右数，下标从 0 开始）强制变成 1：

```js
let n = 5;       // 0101
n = n | (1 << 2) // 1 << 2 = 0100，然后 0101 | 0100 = 0101，第 2 位已经是 1 了
// 如果 n = 3（0011），3 | 4（0100）= 0111 = 7，第 2 位被成功置为 1
```

在 190 题里，`result = result | (n & 1)` 就是用 `|` 把取出来的位"拼"到 `result` 的最低位上。

## 第三个运算符：`^`（按位异或）

`^` 的规则是：**两个对应位不同时结果是 1，相同时结果是 0**。

```js
5 ^ 3
// 5：0101
// 3：0011
// 结果：0110 = 6
```

异或有一个非常神奇的性质，是它区别于其他位运算的核心：

- `a ^ a = 0`，任何数和自己异或都等于 0。
- `a ^ 0 = a`，任何数和 0 异或都等于它自己。

这两个性质组合在一起有一个极其强大的应用——**LeetCode 136（只出现一次的数字）**：给你一个数组，除了一个数字出现一次之外，其他所有数字都出现两次，找出那个只出现一次的数字。用异或，把所有数字全部 `^` 起来，成对的数字会互相消掉变成 0，最后剩下的就是那个只出现一次的数字：

```js
[2, 3, 2, 4, 4].reduce((acc, n) => acc ^ n, 0)
// 0 ^ 2 = 2
// 2 ^ 3 = 1
// 1 ^ 2 = 3
// 3 ^ 4 = 7
// 7 ^ 4 = 3  ← 最终结果是 3，正确！
```

异或还能**不借助临时变量来交换两个数**（虽然现代代码很少这么写，但面试里可能考到原理）：

```js
a = a ^ b  // a 现在存的是 a ^ b
b = a ^ b  // b = (a^b) ^ b = a
a = a ^ b  // a = (a^b) ^ a = b
```

## 第四个运算符：`~`（按位非）

`~` 是一个一元运算符，对每一位取反，0 变 1，1 变 0。

```js
~5
// 5：  00000000 00000000 00000000 00000101
// ~5：  11111111 11111111 11111111 11111010
// 结果是 -6
```

你可能注意到 `~n` 总是等于 `-(n+1)`。这是因为 JavaScript 用**补码**表示负数，按位取反之后符号位也翻转了，结果天然就是 `-(n+1)`。这个性质在实际算法里用途有限，但理解它有助于你看懂别人的代码。

## 第五个运算符：`<<`（左移）和 `>>`、`>>>`（右移）

**左移 `<<`** 的意思是把所有位向左移动 n 位，右边用 0 补齐，左边超出 32 位的部分丢弃。

```js
1 << 3
// 0001 左移 3 位变成 1000 = 8
// 规律：左移 n 位 = 乘以 2 的 n 次方
```

这就是为什么生成"第 k 位是 1，其余是 0"的掩码总是写 `1 << k`——把 1 左移 k 位，就能精确地让第 k 位变成 1。

**有符号右移 `>>`** 向右移动 n 位，但左边补的是**符号位**（正数补 0，负数补 1），保持正负号不变。

**无符号右移 `>>>`** 向右移动 n 位，左边**永远补 0**，不管原来是正数还是负数。

```js
-8 >> 1   // -4，符号位保留，仍然是负数
-8 >>> 1  // 2147483644，把负数当成无符号整数处理，结果是一个很大的正数
```

这就是为什么 190 题里必须用 `>>>` 而不是 `>>`——我们处理的是无符号 32 位整数，不想让符号位的变化影响结果。同理，最后 `return result >>> 0` 是一个常用技巧，把 JS 位运算产生的有符号整数强制转换成无符号表示。

## 把所有运算符串起来记忆

你可以用这张表来建立直觉：

| 运算符 | 名称 | 规则 | 用途 |
|--------|------|------|------|
| `&` | 按位与 | 两个都是 1 才是 1 | 提取某一位、清零某一位 |
| `\|` | 按位或 | 有一个是 1 就是 1 | 设置某一位为 1 |
| `^` | 按位异或 | 不同才是 1 | 消除成对的数、翻转某一位 |
| `~` | 按位非 | 取反 | `~n = -(n+1)` |
| `<<` | 左移 | 右边补 0 | 乘以 2 的幂、生成掩码 |
| `>>` | 有符号右移 | 左边补符号位 | 除以 2 的幂（保留符号） |
| `>>>` | 无符号右移 | 左边补 0 | 处理无符号整数时的右移 |

## 一个思考练习

理解了这些运算符之后，你来想想这段代码在做什么：

```ts
function countOnes(n: number): number {
    let count = 0;
    while (n !== 0) {
        count += n & 1;  // 每次检查最低位
        n = n >>> 1;     // 右移一位，处理下一位
    }
    return count;
}
```

这是 **LC 191（位 1 的个数）**，数一个数的二进制里有多少个 1。你能解释清楚每一步的逻辑吗？想清楚之后，再尝试用 `n & (n-1)` 每次消掉最低位的 1 来写一个更简洁的版本——这两种写法你都掌握了，位运算的基础就打牢了。

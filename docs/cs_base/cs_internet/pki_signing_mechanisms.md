# PKI 签名机制：HTTPS 与 iOS 签名

## 术语表

| 英文缩写/术语 | 全称 | 中文含义 |
|--------------|------|---------|
| **PKI** | Public Key Infrastructure | 公钥基础设施，一整套基于公私钥实现身份验证和安全通信的体系 |
| **CA** | Certificate Authority | 证书颁发机构，负责签发和管理数字证书的可信第三方 |
| **CSR** | Certificate Signing Request | 证书签名请求，向 CA 申请证书时提交的文件（含公钥和身份信息） |
| **HTTPS** | HTTP Secure（HTTP over TLS） | 安全的 HTTP 协议，在 HTTP 基础上加了 TLS 加密层 |
| **TLS** | Transport Layer Security | 传输层安全协议，HTTPS 底层实际使用的加密协议（SSL 的继任者） |
| **SSL** | Secure Sockets Layer | 安全套接层，TLS 的前身，已废弃但名称仍常被混用 |
| **RSA** | Rivest-Shamir-Adleman | 一种非对称加密算法，用于签名和加密 |
| **dSYM** | Debug Symbol | 调试符号文件，用于将崩溃日志还原为可读的代码位置 |
| **IPA** | iOS App Store Package | iOS 应用的分发安装包（本质是 zip） |
| **Keychain** | - | macOS 的密钥链，系统级密码和证书管理工具，私钥存放在这里 |
| **Provisioning Profile** | - | 预置描述文件，规定 app 能装在哪些设备、能用哪些系统权限 |
| **Entitlements** | - | 权限声明，app 声明需要使用的系统能力（推送、iCloud 等） |
| **Team ID** | - | 开发者团队标识，Apple 分配的唯一 ID |
| **Code Signing** | - | 代码签名，用私钥对代码生成数字签名以防篡改和验证身份 |
| **Bundle** | - | 包，macOS/iOS 中一种有固定目录结构的文件夹，.app 和 .appex 都是 bundle |

## 核心概念

### 非对称加密的两种用法

| 操作 | 用什么 | 谁能解开 | 目的 |
|------|--------|----------|------|
| **加密** | 公钥加密 | 私钥解密 | 保密（只有持有私钥的人能看） |
| **签名** | 私钥签名 | 公钥验证 | 防篡改（证明确实是私钥持有者发出的） |

### 证书信任链（PKI）

```
根证书颁发机构（CA）
    │
    │  用 CA 私钥签发
    ▼
开发者/网站的证书 = 公钥 + 身份信息 + CA 签名
    │
    │  用开发者私钥签名
    ▼
最终产物（App / HTTPS 响应）
```

验证时反向走：
1. 产物签名合法吗？ → 用开发者公钥验证
2. 开发者证书是 CA 签的吗？ → 用 CA 公钥验证（CA 公钥内置在设备/浏览器中）

## 一、HTTPS 签名与加密流程

### 目的

安全传输数据：身份验证 + 加密通信

### 证书申请

```
1. 本地生成密钥对（公钥 + 私钥）
2. 生成 CSR（Certificate Signing Request）
   CSR 内容 = 公钥 + 身份信息（域名等） + 私钥签名
3. 将 CSR 发送给 CA（如 Let's Encrypt）
4. CA 验证你的身份（如验证域名所有权）
5. CA 用 CA 的私钥签名 → 生成证书（含你的公钥 + CA 签名）
6. 证书部署到服务器
```

### HTTPS 握手过程

```
浏览器                              服务器
  │                                   │
  │──── 发起连接 ──────────────────────>│
  │                                   │
  │<─── 返回证书（含服务器公钥）─────────│
  │                                   │
  │ 验证证书：                          │
  │ ① CA 签发的吗？                     │
  │   → 用内置 CA 公钥验证 ✓            │
  │ ② 域名匹配吗？ ✓                   │
  │                                   │
  │ 生成随机密钥材料                     │
  │ 用服务器公钥加密密钥材料              │
  │──── 加密后的密钥材料 ──────────────>│
  │                                   │
  │                    用私钥解密 → 得到密钥材料
  │                                   │
  │ 双方根据密钥材料生成相同的对称密钥      │
  │                                   │
  │<═══ 后续所有通信使用对称密钥加密 ═══>│
```

### 加密方式

HTTPS = 非对称加密（握手阶段） + 对称加密（数据传输阶段）

- 非对称加密：安全但慢，仅用于握手时交换密钥
- 对称加密：快速，用于后续所有实际数据传输

### 关键角色

| 角色 | 持有什么 | 做什么 |
|------|---------|--------|
| **CA** | CA 私钥 | 签发证书 |
| **服务器** | 私钥 + CA 签发的证书 | 握手时提供证书，用私钥解密密钥材料 |
| **浏览器** | 内置 CA 公钥（根证书） | 验证证书，用服务器公钥加密密钥材料 |

## 二、iOS App 签名流程

### 目的

身份验证 + 防篡改（不涉及加密通信）

### 证书申请

```
1. Mac 本地生成密钥对（公钥 + 私钥），私钥存在 Keychain 中
2. 生成 CSR（Keychain Access → 从证书颁发机构请求证书）
   CSR 内容 = 公钥 + 身份信息 + 私钥签名
3. 将 CSR 上传到 Apple Developer 后台
4. Apple 用 Apple 的私钥签名 → 生成开发者证书（含你的公钥 + Apple 签名）
5. 下载证书，安装到 Mac
```

### 打包签名过程

```
Xcode 打包时（Mac 上执行）：
    │
    │  用你的私钥对 .app 中每个文件生成签名
    ▼
.app bundle 内容：
├── 可执行文件
├── 资源文件
├── Frameworks/
├── PlugIns/（Extension 的 .appex）
├── _CodeSignature/（签名信息）
└── embedded.mobileprovision（Provisioning Profile）
    │
    │  导出
    ▼
.ipa 文件（本质是 zip，内含 .app）
    │
    │  上传到 App Store Connect
    ▼
分发给用户
```

### iPhone 安装验证

```
iPhone
  │
  │ 下载 .ipa，解压得到 .app
  │
  │ 第一层验证：证书是 Apple 签发的吗？
  │ → 用 iPhone 内置的 Apple 公钥验证证书上的 Apple 签名 ✓
  │
  │ 第二层验证：App 被篡改了吗？
  │ → 用证书里开发者的公钥验证 .app 的签名 ✓
  │
  │ 两层都通过 → 允许安装运行
```

### 关键角色

| 角色 | 持有什么 | 做什么 |
|------|---------|--------|
| **Apple** | Apple 私钥 | 签发开发者证书 |
| **开发者 Mac** | 私钥 + Apple 签发的证书 | 打包时用私钥对 app 签名 |
| **iPhone** | 内置 Apple 公钥（根证书） | 验证证书 + 验证 app 签名 |

### iOS 签名相关概念

| 概念 | 作用 |
|------|------|
| **Certificate（证书）** | 包含开发者公钥 + Apple 签名，证明开发者身份 |
| **Provisioning Profile** | 规定 app 能装在哪些设备、能用哪些权限（推送、Widget 等） |
| **Entitlements** | 声明 app 需要的系统能力 |
| **Team ID** | 开发者账号唯一标识 |

## 三、HTTPS vs iOS 签名对比

| | HTTPS | iOS 签名 |
|--|-------|----------|
| CA 是谁 | Let's Encrypt 等 | Apple |
| 验证方 | 浏览器 | iPhone |
| 私钥在哪 | 服务器上（运行时需要） | Mac Keychain（仅打包时用） |
| 私钥用途 | 握手时解密密钥材料 | 对 app 签名 |
| 目的 | 加密通信 + 身份验证 | 身份验证 + 防篡改 |
| 是否加密数据 | 是（对称加密） | 否 |

---

## 四、安全注意事项

1. **私钥永远不离开本地** — 无论 HTTPS 还是 iOS，私钥都是本地生成、本地保管
2. **不要随便信任根证书** — 抓包工具（Charles/Fiddler）要你安装根证书，本质是插入自己的 CA，可以伪造任何证书实现中间人攻击
3. **换 Mac 要导出私钥** — 私钥丢失意味着无法再给同一个 app 签名，需要重新申请证书
4. **证书吊销** — CA/Apple 可以随时吊销证书，信任链断裂后已签名的产物将不被信任
